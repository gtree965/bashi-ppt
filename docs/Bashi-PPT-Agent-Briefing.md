## 2. What We Learned From Testing LandPPT

### The context window problem (CRITICAL)
LandPPT makes 6 sequential LLM calls per slide:
1. Generate HTML master template (~6000 output tokens)
2. Analyse template "design DNA" (~3500 output tokens)
3. Analyse image needs (~500 output tokens)
4. Generate search keywords (~1400 output tokens)
5. Generate design guidance (~3300 output tokens)
6. Generate final slide HTML (~12000 input tokens → TRUNCATED)

Step 6 sends 12,048 input tokens into a 12,288 context window, leaving
only 240 tokens for output. The model literally cannot generate anything.
LandPPT retried this 4 times, failing each time with `truncated = 1`.

**Bashi PPT's solution:** ONE LLM call total. ~300 tokens in, ~2500 out.
The PPTX rendering is pure Python code with zero LLM involvement.

### The chart hallucination problem
When not explicitly told to skip charts, Qwen3.5-4B generates
fabricated `chart_config` objects on every slide with invented data like
"概念掌握度 85%, 实践操作率 70%". The system prompt MUST include:
"不要生成chart_config或任何图表数据"

### The model's actual performance on this hardware
```
Qwen3.5-4B-Q4_K_M on RX 590 (Vulkan):
  Model size:        2.51 GiB (fits entirely in 8GB VRAM)
  VRAM used:         ~2572 MiB model + ~384 MiB KV cache + ~568 MiB compute
  Prompt processing: ~219 tokens/second
  Generation:        ~16.9 tokens/second
  8-slide outline:   ~3.5 minutes total (including reasoning tokens)

Qwen3.5-9B on same hardware:
  Partially offloaded to CPU/RAM — very slow, not recommended
```

### What the LLM does well at 4B
- Generating structured JSON outlines ✓
- Following simple format instructions ✓
- Chinese content with appropriate vocabulary ✓
- Logical slide ordering (intro → content → conclusion) ✓

### What the LLM does poorly at 4B
- Generating long HTML pages (context overflow) ✗
- Accurate data/statistics (hallucinates numbers) ✗
- Complex multi-step instructions (loses track) ✗
- Knowing when NOT to add optional features ✗

The developer serves three communities with different sensitivities:
1. **Teaching** — No censorship issues with any model
2. **Church/ministry** — Chinese cloud models (DeepSeek, Qwen API, etc.) may
   refuse or self-censor biblical content touching persecution, house churches,
   church-state issues. Local uncensored models have zero restrictions.
3. **Parent communication** — No censorship issues

The app MUST work fully offline with a local model. Cloud API (OpenRouter,
Mistral, etc.) is an optional enhancement, not a requirement.

The config.py design allows hot-swapping between:
- `http://localhost:1234/v1` (LM Studio, default, zero censorship)
- `https://openrouter.ai/api/v1` (OpenRouter, 300+ models)
- `https://api.mistral.ai/v1` (Mistral, no Chinese censorship)

## 5. Chinese Font Handling (CRITICAL)

python-pptx does not automatically handle Chinese fonts. Without explicit
East Asian font configuration, Chinese characters may render as boxes (□)
or fall back to a Latin font with no CJK glyphs.

### Required code pattern for EVERY text run containing Chinese:

```python
from pptx.oxml.ns import qn

def set_chinese_font(run, font_name="Microsoft YaHei", size_pt=20):
    """Set font for a text run with proper Chinese support."""
    run.font.name = font_name
    run.font.size = Pt(size_pt)
    # Critical: set East Asian font explicitly
    rPr = run._r.get_or_add_rPr()
    rPr.set(qn('a:lang'), 'zh-CN')
    # Add East Asian font element
    ea = rPr.makeelement(qn('a:ea'), {})
    ea.set('typeface', font_name)
    rPr.append(ea)
```

### Font availability by OS:
| OS | Font | Status |
|----|------|--------|
| Windows | Microsoft YaHei (微软雅黑) | Pre-installed |
| Linux | Noto Sans SC | `sudo apt install fonts-noto-cjk` |
| macOS | PingFang SC | Pre-installed |

The renderer should default to "Microsoft YaHei" since the primary
target is Windows. Add a config option for Linux users to switch.

## 6. python-pptx Gotchas

### Slide dimensions must be set explicitly
```python
from pptx.util import Inches
prs = Presentation()
prs.slide_width = Inches(13.333)   # 16:9 widescreen
prs.slide_height = Inches(7.5)
```
Default is 4:3 (10" × 7.5") which is wrong for modern presentations.

### Use blank slide layout
```python
slide_layout = prs.slide_layouts[6]  # Blank layout
slide = prs.slides.add_slide(slide_layout)
```
Layout indices vary by template. Layout 6 is typically "Blank" in the
default template. Always verify. Using a blank layout and adding textboxes
manually gives full control over positioning.

### Background color
```python
from pptx.dml.color import RGBColor
from pptx.oxml.ns import qn

background = slide.background
fill = background.fill
fill.solid()
fill.fore_color.rgb = RGBColor(0xFF, 0xFF, 0xFF)  # White
```

### Bullet points in textboxes
```python
tf = txBox.text_frame
tf.word_wrap = True
for i, point in enumerate(content_points):
    if i == 0:
        p = tf.paragraphs[0]
    else:
        p = tf.add_paragraph()
    p.text = point
    p.level = 0
    p.space_after = Pt(8)
    set_chinese_font(p.runs[0], size_pt=20)
```

### Don't use `p.text =` and then try to format
Setting `p.text` creates a single run. You must access `p.runs[0]`
AFTER setting text to apply font formatting. Or create a Run explicitly:
```python
from pptx.oxml.ns import qn
run = p.add_run()
run.text = "要点内容"
set_chinese_font(run, size_pt=20)
```
